## مشکلات رفع‌شده (Fixed)

### 0e. فضای زیاد فیلترها در موبایل
- **راه‌حل:** `v2.9.23` — دو bottom-sheet موبایل با تریگر Elementor  
  - تریگر: `wclf-open-filters` / `wclf-open-sorting`  
  - منبع: `wclf-mobile-filters-source` / `wclf-mobile-sorting-source`  
  - راهنما: `Docs/Mobile-Sheets.md` + صفحه ادمین

### 0d. تکون خوردن صفحه به‌خاطر حذف اسکرول‌بار در پریلودر
- **علائم:** با آمدن پریلودر اسکرول‌بار حذف می‌شد و viewport جابه‌جا می‌شد.
- **رفع:** `v2.9.22` — دیگر `overflow: hidden` ست نمی‌شود؛ اسکرول با `wheel` / `touchmove` / `keydown` قفل می‌شود و اسکرول‌بار سر جایش می‌ماند

### 0c. جابه‌جایی viewport به راست هنگام نمایش پریلودر
- **رفع اولیه (ناقص):** `v2.9.21` — جبران با padding (جایگزین شد با 2.9.22)

### 0. Overflow افقی هنگام پریلودر تمام‌صفحه
- **رفع:** `v2.9.19` — حذف `100vw`، قفل overflow روی html/body

### 0b. پیام خالی وقتی فیلتر نتیجه‌ای ندارد
- **علائم:** کاربر با فیلترهای سخت به صفحهٔ خالی بدون راهنمایی می‌رسید.
- **رفع:** `v2.9.20` — پیام «محصولی با این فیلترها پیدا نشد» + دکمه «پاک کردن فیلترها» (AJAX و لود اولیه)

### 1. آیکن سورت هاردکد به دامنه خارجی
- **علائم:** آیکن مرتب‌سازی از `salamatfarivar3.faramoujdev.ir` لود می‌شد و روی سایت‌های دیگر می‌شکست.
- **رفع:** `v2.9.11` — فایل محلی `assets/sort-from-top-to-bottom.svg`

### 2. فیلتر دسته روی آرشیو برند / تگ / جستجو خالی بود
- **علائم:** برخلاف مستندات، در آرشیو برند/تگ/جستجو لیست دسته خالی برمی‌گشت.
- **رفع:** `v2.9.12` — `get_scoped_product_cat_terms_for_page()`

### 3. فیلتر ویژگی contextual نبود
- **علائم:** همه ترم‌های سایت نمایش داده می‌شد، نه فقط ترم‌های مرتبط با محصولات صفحه.
- **رفع:** `v2.9.13` — `get_contextual_attribute_terms()` + `exclude_attribute` در Query Helper

### 4. دکمه «همه» همه فیلترها را پاک می‌کرد
- **علائم:** کلیک روی «همه» در مرتب‌سازی، قیمت/دسته/برند و… را هم حذف می‌کرد.
- **رفع:** `v2.9.14` — فقط `orderby` پاک می‌شود

### 5. `orderby=discount` محصولات بدون تخفیف را حذف می‌کرد
- **علائم:** مرتب‌سازی تخفیف عملاً فیلتر هم بود.
- **رفع:** `v2.9.15` — فقط سورت با named meta clause؛ محصولات بدون تخفیف باقی می‌مانند

### 6. وابستگی سخت به ووکامرس وجود نداشت
- **علائم:** بدون WooCommerce پلاگین بدون هشدار بوت می‌شد.
- **رفع:** `v2.9.16` — `Requires Plugins: woocommerce` + پیام ادمین

### 7. مستندات روی 2.9.5 مانده بود
- **علائم:** PRD / Architecture / AI-AGENTS با کد هم‌خوان نبودند.
- **رفع:** `v2.9.17`

### 8. کش صفحه / CDN می‌توانست HTML فیلترشدهٔ کهنه سرو کند
- **علائم:** AJAX نتایج قدیمی یا اشتباه برمی‌گرداند.
- **رفع:** `v2.9.18` — `DONOTCACHEPAGE` + هدرهای `no-store` + `fetch(cache: 'no-store')`

### 9. بلور فقط روی ناحیه محصولات بود (درخواست محصول)
- **رفع:** `v2.9.18` — overlay تمام‌صفحه + پریلودر روی آن

### 10. Overflow افقی هنگام نمایش پریلودر / بلور (دسکتاپ و موبایل)
- **علائم:** هنگام لود فیلتر، صفحه اسکرول افقی می‌خورد؛ قبلاً در موبایل هم دیده شده بود (`v2.9.6`).
- **علت ریشه‌ای:** استفاده از `width/height: 100vw/100vh` برای overlay تمام‌صفحه — `100vw` اغلب عرض اسکرول‌بار را هم حساب می‌کند و باعث overflow افقی می‌شود.
- **رفع:** `v2.9.19`
  - overlay با `top/right/bottom/left: 0` بدون `100vw`
  - قفل `overflow` / `overflow-x` / `overflow-y` روی `html` و `body`
  - کلاس `wclf-ajax-loading` + `overscroll-behavior: none` + `touch-action: none`

## مشکلات باز / نکات باقی‌مانده (Open / Watch)

### A. ID تکراری برای چند فیلتر ویژگی
- همه شورت‌کدهای attribute از `id="attributeFilterWrapper"` استفاده می‌کنند.
- چند attribute در یک صفحه = HTML نامعتبر؛ swap بر اساس index شکننده است.
- **پیشنهاد:** ID یکتا مثل `attributeFilterWrapper-{slug}` و آپدیت سلکتورهای JS

### B. نام‌گذاری ناسازگار شورت‌کدها
- `elementor_*` / `beban_product_filters` / `wclf_*`
- فقط بدهی فنی؛ عملکرد را خراب نمی‌کند

### C. استایل فیلترها داخل پلاگین نیست
- `assets/css/` خالی است؛ ظاهر وابسته به تم است

### D. Plugin URI هنوز placeholder است
- `https://example.com/`

### E. bulk recalculate تخفیف بدون batch
- روی کاتالوگ خیلی بزرگ ممکن است timeout بخورد

### F. `:has()` برای مخفی کردن ویجت‌های خالی Elementor
- مرورگرهای خیلی قدیمی ممکن است wrapper را جمع نکنند

## چک‌لیست تست دستی (بعد از تغییر overlay)

1. دسکتاپ: فیلتر قیمت/دسته را بزن → صفحه اسکرول افقی نگیرد
2. موبایل (Chrome/Safari): همین سناریو؛ نوار آدرس و gesture هم چک شود
3. بلور تمام‌صفحه دیده شود و پریلودر مرکز روی آن باشد
4. بعد از اتمام AJAX، اسکرول صفحه به حالت قبل برگردد
5. با فعال بودن کش صفحه (LiteSpeed / WP Super Cache / Cloudflare)، URLهای فیلترشده نتیجهٔ کهنه ندهند
